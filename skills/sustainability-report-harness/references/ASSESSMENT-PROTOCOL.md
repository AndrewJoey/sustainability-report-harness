# Assessment Protocol

Use this protocol for requirement-level response assessment.

Each selected requirement must receive one status: `fully_addressed`, `partially_addressed`, `not_addressed`, `not_applicable`, or `needs_confirmation`. Record linked content and evidence, rationale, missing information, improvement suggestion, confidence, confidence reason, and review status.

Require a human-confirmed reason for `not_applicable`. Explain low confidence. Keep peer position in
an independent `peer_assessments` collection, allow only `peer_reference` evidence there, and never
score actual ESG performance.

Agent-created standards and peer assessments remain `unreviewed` until a named reviewer accepts,
rejects, or edits them. Deterministic code validates contracts and evidence classes; it does not make
semantic response judgments.
