# Assessment Protocol

Use this protocol for requirement-level response assessment.

Each selected requirement must receive one status: `fully_addressed`, `partially_addressed`, `not_addressed`, `not_applicable`, or `pending_confirmation`. Record linked content and evidence, rationale, missing information, improvement suggestion, confidence, confidence reason, and review status.

Require a human-confirmed reason for `not_applicable`. Explain low confidence. Keep peer position separate from standards response status and never score actual ESG performance.

M1 validates Assessment records but does not make semantic response judgments.
