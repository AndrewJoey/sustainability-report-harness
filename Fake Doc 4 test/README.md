# Fake Doc 4 test

这是一套用于 Sustainability Report Harness 的**完全虚构测试资料**。企业、人员、地点、数值、目标和同行案例均为模拟内容，不对应任何真实主体，也不得用于监管、审计、鉴证或投资判断。

## 虚拟企业

- 企业：澄湾循环科技（苏州）有限公司（英文简称：ClearBay Circular Technologies，简称“澄湾科技”）
- 行业：再生材料与循环包装制造
- 报告期：2025-01-01 至 2025-12-31
- 报告类型：可持续发展报告
- 报告边界：苏州总部及工厂、无锡仓储中心；不含少数股权合营企业和员工共享办公点
- 目标语言：简体中文

## 文件用途

```text
sources/client/
  01_公司概况与治理.docx              企业边界、组织、治理职责、商业模式
  02_可持续发展政策与目标.docx        政策承诺、2025 进展、2030 目标
  03_气候风险与韧性评估.pdf          可搜索 PDF；风险、情景、应对措施
  04_利益相关方与重要性评估.pdf      可搜索 PDF；参与方式、重要议题和排序
  05_环境绩效数据.xlsx               能源、排放、水、废弃物及公式口径
  06_员工与安全数据.xlsx             员工结构、培训、职业健康安全数据
  07_供应链与商业道德.docx           供应商管理、举报、合规和事件数据
  08_2024可持续发展回顾.docx         既有报告，用于测试结构与文风保留

sources/peer/
  01_同行优秀案例_环屿材料2024.pdf    完全虚构同行案例；只能用于风格/质量对标

sources/requirements/
  01_报告任务书与内容模板.docx        报告用途、受众、目录、文风和交付要求

standards/
  simulated-standard-a.json           仓库自带模拟准则副本
  simulated-standard-b.json           仓库自带模拟准则副本
```

根目录还包含：

- `project.yaml`：项目配置草案；准则尚未锁定，避免绕过人工确认。
- `intake_proposal.json`：可供 `confirm_intake.py` 使用的对话 intake 回答文件。
- `DATA_NOTES.md`：统一口径、关键勾稽、已知缺口和测试注意事项。

## 推荐测试方式

1. 用 Harness 在此目录或另一个新项目目录建立项目骨架。
2. 先确认数据授权，再使用 `intake_proposal.json` 完成 intake。
3. 由测试人员明确确认并锁定两套 `standards/` 下的模拟准则；它们不是正式准则。
4. 解析 `sources/client/` 和 `sources/peer/`。
5. 检查 Word 段落/表格定位、PDF 页码定位、Excel 工作表/单元格/公式保留。
6. 同行案例不得成为澄湾科技事实证据。

## 明确边界

- 所有客户材料均为模拟事实，但在本测试项目内应按 `client_evidence` 处理。
- 同行材料均为模拟参考，只能按 `peer_reference` 处理。
- 2025 年温室气体数据为内部盘查结果，未获第三方鉴证。
- 范围三仅覆盖采购商品与服务、上游运输及废弃物处理，不代表完整价值链盘查。
- 财务影响为管理层内部敏感性估算，不是财务预测。
- 不含扫描 PDF，因此标准流程不应触发 OCR 阻塞。
